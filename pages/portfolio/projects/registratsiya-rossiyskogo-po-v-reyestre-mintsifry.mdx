# Регистрация российского ПО в реестре Минцифры

<Steps>
### Описание ситуации

Компания, в которой я работал, разрабатывала сервисы искусственного интеллекта в медицинских целях. Для участия в госзакупках и получения преференций от государства наши сервисы требовалось регистрировать в реестре Минцифры. Для этого были привлечены внешние консультанты, имеющие релевантный опыт.

### Задача

Мне поручили помогать консультантам в заполнении заявлений, а также составлять необходимые документы.

### Исполнение

Заявления мы заполняли на сайте Госуслуг, подписывали их усиленной ЭЦП. К заявлениям прикрепляли файлы с документами. Вот некоторые из тех, над которыми я работал:

- Руковоство по эксплуатации ПО
- Описание процессов жизненного цикла ПО
- Описание функциональных характеристик ПО
- Образец лицензионного договора
- Технические условия

При выполнении задачи руководствовался действующими нормативами и официальной документацией. Например:

- [Единый реестр российских программ для ЭВМ и БД. Официальный сайт реестра ↗](https://reestr.digital.gov.ru)
- [Классификатор программ для ЭВМ и БД. СПС Гарант ↗](https://base.garant.ru/74832091/)
- [Методические рекомендации по подаче заявлений и уведомлений. Документ pdf ↗](https://reestr.digital.gov.ru/upload/documents/Методичка_Подача_заявления.pdf)
- [Правила формирования и ведения единого реестра программ для ЭВМ и БД. СПС Гарант ↗](https://base.garant.ru/71252170/)
- [Требования к правообладателям и программам. Официальный сайт Госуслуг ↗](https://www.gosuslugi.ru/reestrpo)

### Результат

На момент моего увольнения из компании один из сервисов внесен в реестр ([ссылка на реестровую запись ↗](https://reestr.digital.gov.ru/reestr/1910660/)). По другим не знаю.

</Steps>
