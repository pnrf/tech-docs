# Nextra 2.x

<Callout type="warning">
  Ниже представлен довольно фривольный перевод документации Nextra 2.x на русский язык. Информация может быть устаревшей. Лучше пользоваться официальной документацией на [Nextra.site](https://nextra.site/docs)

**Nextra** – это фреймворк на основе Next.js, который позволяет создавать статические сайты с документацией. Система поддерживает все основные возможности Next.js, а также разметку Markdown для генерации контента.

## Популярные вопросы

`Nextra FAQ` представляет собой подборку ответов на наиболее часто задаваемые вопросы. Если вы не нашли среди них интересующую вас информацию, вы можете обратиться к разработчикам напрямую. Для этого, пожалуйста, [начните обсуждение](https://github.com/shuding/nextra/discussions).  

<br />

export function FAQBox({ title, children }) {
  return (
    <details
      open
      className="last-of-type:mb-0 rounded-lg bg-neutral-50 dark:bg-neutral-800 p-2 mt-4"
    >
      <summary>
        <strong className="text-lg">{title}</strong>
      </summary>
      <div className="nx-p-2">{children}</div>
    </details>
  )
}

<FAQBox title="Возможно ли в Nextra использовать Next.js `app router`?">
	Нет. Nextra работает только через директорию `/pages`. Поддержка `app router` пока еще нами не реализована. Однако вы можете использовать `/app` и `/pages` одновременно. Для этого поместите документы внутри `/pages`, а другие роуты – в `/app`.
</FAQBox>

<FAQBox title="Возможно ли использовать X вместе с Nextra?">
  В большинстве случаев "да". Поскольку Nextra представляет собой плагин для Next.js, практически все, что может быть реализовано с помощью React, может быть применено в Nextra. Вот несколько примеров:

- [Применение Tailwind CSS](../knowledge-base/nextra/guide/advanced/tailwind-css)
- [Применение пользовательских CSS и Sass](../knowledge-base/nextra/guide/custom-css)
- [Применение пользовательских шрифтов](https://nextjs.org/docs/pages/building-your-application/optimizing/fonts)

</FAQBox>

<FAQBox title="Как добавить в Nextra компонент live coding?">
  Для этого вы можете воспользоваться сторонними библиотеками. Например, [Sandpack](https://sandpack.codesandbox.io) и [react-live](https://github.com/FormidableLabs/react-live). Они позволяют добавлять live coding компоненты в MDX.
</FAQBox>
